# 📡 天线匹配软件 (Antenna Matching Tool)

Murata LC元件库 + 多频段效率评估 + 多拓扑优化

> A tooling for antenna matching, powered by Python+FastAPI backend and React frontend.

## 一键启动

### Windows（双击即可）
- **`启动.bat`** — 生产模式 (推荐，单窗口，浏览器自动打开)
- **`启动-开发模式.bat`** — 开发模式 (前后端分离，支持热更新)

### 通用方式（Windows/macOS/Linux）
```bash
# 生产模式
python start.py

# 开发模式 (需要 Node.js)
python start.py --dev
```

启动后浏览器打开：**http://127.0.0.1:8000**。首次运行若前端构建产物不存在，会自动执行 `npm ci` 和 `npm run build`。

开发环境安装：

```bash
python -m pip install -r requirements-dev.txt
```

## 功能特性
- **多拓扑优化**: L-Network / Pi-Network / T-Network / Ladder-N
- **多频段效率评估**: 指定多个频段，评估匹配后全频段效率
- **S11曲线对比**: 匹配前后S11全频段对比图
- **Smith Chart**: 阻抗匹配可视化
- **手动调谐**: 手动组合理想/实测 L/C、R、有损传输线和开路/短路 Stub；提供可点击的 DUT→元件→50 Ω 物理信号链、对数连续调谐、E24 标准值快调、同页目标频段编辑、可配置回损目标、逐频段最差点/裕量检查和 S11 目标频段覆盖层；固定当前拓扑的智能局部优化可选择最差点/平均/均衡目标，连续精修理想值及传输线参数，同时锁定实测 S2P 料号，支持后台进度、取消与一键撤销；优化完成后按每个旋钮 ±10% 物理复算生成灵敏度排行，并在 S11/Smith 上叠加优化前轨迹，便于解释改善来源与识别高风险参数；未达到门限时定位各频段最差频点及复阻抗，并在 DUT 侧和端口侧物理粗探串/并联 L、C；每个有效方向再从活动目录选取最接近的最多 5 个真实料号加载 S2P 粗筛，前两名必须在“全部 DUT 频点＋频段采样点”联合网格上再次通过才作为可制造候选，可分别一键应用理想方向或锁定精确实测料号；当前精确网络可运行可取消的确定性 Monte Carlo 制造良率分析，混合理想元件、真实 S2P 和传输线公差，显示通过率 Wilson 置信区间、P5 最差回损及最差样本风险元件；标准阻抗 Smith 图包含等电阻/等电抗网格、回损目标圈、Γ/VSWR/输入阻抗读数，并可点击轨迹选择最近频点；支持按标称值自动选料或锁定精确料号、逐激励端口独立网络/撤销会话、DUT 向信号源的物理级联顺序拖拽/上下移动、S11 曲线拖动选频和功率闭合；仅在进入工作区后加载目录和计算，并用 DUT 文件名与网络语义 SHA-256 拒绝 CST 切换期间的过期结果；可命名冻结最多 12 个带频段/门限/逐频段证据的方案，工程对比矩阵显示最差频段、目标裕量、频点指标、相对当前网络的 BOM 增删改和证据来源，选择最多 4 个方案在当前 DUT 上物理重算并叠加 S11/Smith，并随完整性校验的工程与 HTML/PDF 报告保存/恢复
- **传输线自动综合**: 有界搜索 Z0、电长度和 line/Stub 顺序，支持后台进度、取消、功率守恒与候选精确扫频
- **PCB 微带制造模型**: 根据板材 Dk/Df、介质厚度、铜厚/粗糙度和线宽规则生成实际走线尺寸，并以频散和物理损耗重新排名
- **EM/VNA 布局块联合综合**: 从 SNP 目录选择二端口 S2P，按 DUT 侧或连接器侧固定级联到每个传输线候选；支持端口翻转、系统 Z0 重归一化和左右 fixture 去嵌入，界面显示被动性/条件数/重级联残差，项目与报告复核所有文件 SHA-256
- **CST 结果闭环**: 可通过 CST 2025 官方 Python 运行时发现打开工程和 S 参数结果树，一键直出、严格校验并载入 DUT；求解期间每 2 秒自动刷新状态，同一工程重复导出更新稳定文件名并记录前后原始 SHA-256，同时用频率、完整复数 S 矩阵和 Z0 的语义指纹识别真实电气修订；仅注释/格式变化时保留优化候选、端口配置和手动调谐工作区，不覆盖来源不明的同名文件
- **端口配置**: 支持多端口DUT (open/short/load/component)
- **联合多端口优化**: 考虑S21耦合的联合匹配
- **定向隔离硬约束**: 可在界面定义任意 `S_destination,source` 的频段上限，候选先按约束通过状态、再按效率排序
- **实测器件拓扑综合**: 每端口自动选择 0–6 个元件，使用 Coilcraft 0402HP / Murata GQM18 的完整 S2P 寄生与损耗模型进行联合精修
- **逐端口工程拓扑约束**: 固定 LC 模式可为每个端口独立选择自动、裸 DUT、单元件、串联、并联、L 型或自定义拓扑白名单；约束在理想种子和真实料号搜索前生效，并随诊断与项目请求保存
- **可采购器件目录约束**: Component Series 面板从完整元件库发现文件夹级系列，可按厂商、系列或封装搜索并选择，并进一步约束厂商、封装、最大容差、料号额定电压代码和介质类型；预览会显示实际剩余 L/C 数量，缺失元数据可选择“保留并标记”或“严格排除”。电压/介质推断不会冒充厂商验证值；筛选配置、来源统计、有效数量和 SHA-256 指纹随候选、项目及 HTML/PDF 报告保存
- **深层匹配网络搜索**: 每端口可扩展到 6 个真实 S2P 元件；5–6 元件采用可取消的渐进拓扑束搜索，高质量小目录基准对 8192 个 exhaustive 候选达到 top-10 精确料号与拓扑 100% 召回
- **功率平衡分析**: 反射/耦合/元件损耗/辐射占比可视化
- **Murata元件数据库**: 29K+ 真实元件S参数，支持快速查询
- **辐射效率集成**: 支持每端口独立η_rad文件加载
- **多目标优化**: 6种优化模式 (效率/回损/平衡/最差/平均/低成本)
- **多场景综合匹配**: 多个天线 SNP 测量文件共用同一拓扑和同一组离散 S2P 元件，使用核心 dB 目标与完整物理功率求解器，按加权平均、最差场景或平衡目标求综合最优解；中文工程界面提供频段/权重预检、搜索阶段、物理评估数、耗时和协作式取消
- **按需工作区初始化**: 首屏完成健康检查和数据源配置后才开放实测目录请求；多场景的 L/C、效率文件和拓扑仅在首次进入时加载，同一数据源版本切换页面不重复请求；刷新 SNP 列表不再重配元件库或清空优化会话
- **多场景完整目录认证**: 官方三种阻抗状态在完整 43 个 Coilcraft 0402CS + 882 个 Murata GJM15 模型上先以连续理想综合分配预算、物理粗筛全部 16 个二元件拓扑，再做全目录邻域精修；45 秒 Balanced 搜索后用 82 个独立频点重排，参考拓扑第 1、参考 5.6 pF / 4.3 nH 网络第 2，最佳最差效率与实测核心基线只差 0.0023 dB
- **MDIF 可调器件物理评估**: 读取多状态 MDIF 实测模型，为每个多频段配置自动选择状态，并在共享固定 S2P 网络下报告 total efficiency、器件损耗与功率守恒
- **可调网络自动综合**: 联合搜索固定拓扑、实测料号和各 configuration 的 MDIF 状态；官方 variable-C 案例可自动召回串联 2.8 pF + 15 nH 与 8/2/1 pF 状态映射
- **后台进度与真实取消**: 单端口、多端口、多场景、tuner、switch 与传输线自动综合统一以后台 job 运行；界面显示理想拓扑预筛、实测 S2P 展开、联合排名和全物理复核阶段。取消会触发核心协作式检查，并保留上一次已完成结果或取消前已经完成且功率守恒的部分候选
- **Anytime 时间预算**: 六元件真实 S2P 搜索在软截止时保留已经完成的候选；更长预算沿用同一确定性搜索主干，只增加离散邻域和精修，不遗忘短预算好解
- **单/多端口数值核心收敛**: 产品与旧协议入口统一委托全频带实测求解 service；全局兼容白名单与逐端口白名单都由 measured core 在种子、深层发现和联合排名阶段强制执行，旧求解路由仅为外部脚本兼容保留并标记 deprecated，Web 不再调用
- **裸 DUT 权威基线**: 单端口和多端口零元件请求都由 rfmatch_core 在所有带内频点计算反射、耦合、吸收和功率守恒，无需加载元件库，也不再切换到历史求解器；报告明确区分裸 DUT 核心基线与实测元件搜索
- **部分目录自适应**: 单端口和全矩阵多端口 measured core 按当前目录实际拥有的 L/C 类型裁剪拓扑；只有电感或只有电容的专业小目录仍可执行对应拓扑综合，缺少另一类型不会导致整个搜索切换实现
- **权威 Touchstone 数值入口**: 产品与 rfmatch_core 共用严格解析器，覆盖多端口 RI/MA/DB、DC 起始点、行折返、内联注释、Touchstone 2.0 Network Data、Full/Lower/Upper 矩阵、`12_21/21_12` 二端口顺序、三端口以上规范行顺序、跨行逐端口正实数参考阻抗及非 50 Ω 功率求解；不支持的参数域、mixed-mode、复数参考、不完整记录、频点数不符、负频率和非递增频率会明确拒绝
- **追加预算续跑**: 截断结果显示停止阶段和评估数，可一键通过可取消后台任务增加预算；单端口和多端口联合实测搜索复用已加载 S2P、理想拓扑前沿、逐端口子问题及精确评估缓存，旧/新网络合并排序保证已找到方案不会因续跑消失；不支持检查点的路径明确回退为确定性重跑
- **真实目录续算认证**: 官方 Optimization Settings DUT 与 371 个 0402HP/GQM18 模型上，8 秒部分解追加预算后与 20 秒冷启动得到完全相同的拓扑、得分和 835 次唯一物理评估；追加阶段耗时约为冷启动的 40%
- **多端口续算认证**: 官方三天线耦合 DUT 与 371 个真实模型已覆盖每端口 0–1 和 0–2 元件完整前沿；0–2 案例续算后与冷启动得到相同拓扑、得分和 722 次唯一物理评估，追加阶段耗时约为冷启动的 26%
- **完整目录自动拓扑召回**: 不提供拓扑白名单时，超深度搜索以理想组合筛选、端口×拓扑多样性、单端口拓扑邻域和深度增长精修自动恢复 `SCPL / PCSL / PCSL`；官方 145 型号案例中正确拓扑第 1、精确 BOM top-20，优化后产品仅在三端口以内且预算至少 150 秒时启用该档位
- **自动深搜产品入口认证**: 统一 `run_tuning_joint` 端到端加载 145 型号且不传拓扑提示，在 125.62/150 秒内无截断完成；正确拓扑第 1、精确 BOM 第 15、4601 次物理评估、69 个懒加载模型，证明 UI/API 档位不是只存在于核心基准
- **辐射效率闭环认证**: 官方三频段 DUT 与 51 点 radiation-efficiency 曲线已进入统一单端口目标；带内平均/最差及跨频段权重可由 API/UI 显式设置，连续拓扑先验的 L Q/参考频率及 L/C ESR 也可配置且不会冒充最终实测器件性能。完整 41 个 Coilcraft 0402HP + 104 个 Murata GQM18 实测模型对照中，按教程 Q=50@1 GHz、C ESR=0.3 Ω 重放后，效率感知网络在 65 个独立频点上的最差总效率比纯匹配网络提高 0.356 dB（官方通用器件教程约 0.3 dB），目标得分提高 0.052 dB，功率误差为 0
- **多路开关自动综合**: MDIF 支持分类状态和任意 N 端口 ACDATA；SP2T/SP3T 保留插损、隔离和 throw 耦合，自动搜索支路类型、0–2 元件输入块和状态映射，再吸附到 0402CS/GJM15 实测 S2P；各复杂度最优解均可独立选择、比较带内曲线并随项目保存恢复，同时支持后台进度与真实取消
- **综合手动调试**: 指定 1–4 个元件及拓扑，每个位置可切换理想 L/C 或真实离散料号，同时查看各场景 S11 / total efficiency
- **效率数据语义**: 每个场景可不提供效率，或提供 radiation efficiency / total efficiency；total efficiency 会依据原始 S11 反推辐射效率后重新评估匹配网络
- **项目保存与恢复**: 顶部 `Projects` 可保存输入哈希、优化配置、元件库指纹、候选方案与选择状态；打开时校验原始 Touchstone SHA-256
- **BOM 与替代料闭环**: 项目管理器可导出带数量、端口位置、采购参数及来源标签的 UTF-8 CSV BOM；方案中的实测料号可直接运行频带内替代料分析，按完整二端口 S 矩阵 RMS/最大差异和小权重标称值偏差排序，并显示实际比较频点、失败数和目录截断状态
- **可追溯 HTML 报告**: 每个完整性校验通过的项目可一键导出自包含报告，包含软件/核心版本、输入哈希、候选比较、选中 BOM、配置状态、功率守恒和分频段 SVG 曲线，并支持浏览器打印为 PDF

## 多场景综合匹配

启动后点击顶部 **多场景联合**：

1. 勾选至少两个同端口数的 SNP 测量场景，可分别设置权重和效率文件。
2. 选择元件数量及一个或多个候选拓扑，配置频段和综合目标。
3. 运行后得到共享料号的排序结果、逐场景最差指标及跨场景曲线。
4. 点击结果中的 **转到手动调谐** 可将该解带入手动模式，逐位置替换真实 S2P 元件或理想值并重新评估。

联合优化通过后台任务运行。界面持续显示理想拓扑预筛、实物器件预筛、实物器件精搜和独立密集验证阶段，并报告耗时与物理评估次数；深度搜索可随时取消，不会再由一个 15–150 秒的长 HTTP 请求阻塞页面。运行前会拒绝反向/空频段、负权重、全零权重、无拓扑和无效端口。

手动模式采用实时工作流：修改拓扑、元件料号、理想 L/C 值、频段、效率文件或场景后，会自动对所有已选 SNP 应用同一网络，并同步刷新全部 S11 与 total-efficiency 曲线。高级设置中可选择 41、81 或 161 个实时曲线采样点。

自动多场景搜索使用同一组命名质量档位。稀疏搜索网格只用于候选发现；返回前会在独立的
21/41/81 点每频段网格上重新计算并重排，避免少量频点上的偶然赢家直接进入结果。诊断同时
公开连续理想拓扑排名、全拓扑物理粗筛、目录邻域精修、模型缓存和最终验证的实际评估量。

自动综合提供四个后端权威质量档位：`Quick`（15 s / 8 输出 / 3 点）、`Balanced`
（45 s / 10 输出 / 5 点）、`Thorough`（120 s / 10 输出 / 10 点）和 `Automatic topology deep`
（150 s / 50 输出 / 3 点）。最后一档仅对 Fixed LC、2–3 端口、每端口不超过两元件且所有
端口保持自动拓扑的请求启用完整目录拓扑发现；其他请求会明确显示实际采用的一般分层路径。
修改任一底层数字会进入 `Custom`，不会继续冒充命名档位。后台任务显示已用/总预算、当前阶段
与执行策略；候选、项目和 HTML/PDF 报告保存请求档位、实际计划和校准证据。继续优化会转成
累计 `Custom` 预算，避免原命名预设在恢复时把追加秒数重置。

## 配置

默认从 `data/snp/` 与 `data/Murata/` 读取数据。可用以下环境变量覆盖：

- `RFMATCH_SNP_DIR`
- `RFMATCH_MURATA_DIR`
- `RFMATCH_DB_PATH`
- `RFMATCH_PROJECTS_DIR`（默认 `artifacts/projects/`）

## 依赖

- Python 3.10+（numpy、fastapi、uvicorn、pydantic）
- Node.js 18+（开发模式或首次构建前端时需要）

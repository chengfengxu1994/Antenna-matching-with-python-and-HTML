# RF Matching 架构说明

## 当前运行结构

```text
apps/web (React / Vite)
    │  /api
apps/api (FastAPI)
    ├─ api/models.py       请求契约
    ├─ api/state.py        本地单用户会话与路径配置
    ├─ engine/             当前 Web 产品使用的算法与元件库适配
    │  ├─ component_environment.py 精确料号 tempco/偏置 sidecar 与证据校验
    │  ├─ snp_monitor.py      CST/HFSS 导出稳定性监视与严格 Touchstone 交接
    │  ├─ snp_provenance.py   与文件 SHA 绑定的求解器/VNA 来源持久化
    │  └─ cst_bridge.py       CST 官方 Python 运行时隔离、工程与结果树发现
    ├─ project_store.py    版本化、原子写入、完整性校验的项目快照
    └─ Murata DB / S2P / SNP

packages/rfmatch-core/     独立、可测试的下一代数值内核
    ├─ components.py       文件/ZIP/SQLite 懒加载实测模型契约与模型哈希
    ├─ physical_optimizer.py 理想引导、实测料号精修与有界邻域搜索
    ├─ search_validation.py 单/多端口小目录 exhaustive、精确料号/拓扑 top-k 召回率校准
    ├─ physical.py         多端口节点图、实测 S2P 嵌入与功率守恒
    ├─ transmission_line.py 有损均匀传输线、开路/短路 Stub 与 S2P 导出
    ├─ line_optimizer.py   Z0/电长度有界多起点搜索、7 类线/Stub 物理拓扑与取消/超时
    ├─ microstrip.py       PCB 基板、Hammerstad/Kirschning 几何频散、损耗与线宽反解
    ├─ tolerance.py        可复现 Monte Carlo、L/C 系统偏置、批次相关/温度模型、Wilson 区间与良率排序
    ├─ state_tolerance.py  开关/MDIF 多状态共享器件及共享环境联合良率
    ├─ optenni_opr.py      Optenni 4.x OPR XML 的只读项目证据提取
    └─ tolerance_golden.py Optenni 原生容差导出解析与判据复算
```

产品当前是本机单用户应用。`AppState` 保存当前 DUT、元件库、效率数据和最近结果，因此不应直接以多进程或公网服务方式部署。

## 模块边界

- `apps/web/src/components`：工作流与展示，新业务状态逐步迁移到页面级 hooks。
- `apps/api/api`：请求校验、错误映射和响应组装。
- `apps/api/engine`：现役产品引擎，不引用 HTTP 对象。
- `packages/rfmatch-core`：参考数值契约和回归基线，逐步替代现役引擎的重复能力。
- `apps/api/engine/tuning_service.py`：产品适配层；把目录、ZIP、SQLite 元件统一成
  核心懒加载模型，将当前候选转换为物理拓扑，并调用统一搜索与良率分析。
- `/api/manual-tune` 已不再维护独立的 L/C 公式；理想/实测 L/C、理想 R、参数化
  传输线和开短路 Stub 均转换为同一个 `rfmatch_core` 节点图，返回逐频器件损耗与
  功率守恒误差。直通线固定为 series，Stub 固定为 shunt，避免无物理意义的组合。
  请求可携带精确实测料号；非空料号必须在当前目录内精确命中，空值才允许按标称值
  自动选取最近器件。请求还绑定活动 DUT 文件名与可选网络语义 SHA-256，服务端不接受
  CST 切换或重新求解后到达的旧版本请求，响应回传同一 DUT identity 供前端二次校验。
- Web 手动调谐页按激励端口分离保存当前网络及各自的撤销/恢复栈；其他端口
  仍按显式 load/open/short 状态端接，因此界面不会向已端接端口发送元件。E24 步进、
  对数连续滑杆、S11 点击/拖动游标和 Smith 目标标记只改变前端交互。元件数组按 DUT 向外定义物理
  级联顺序；拖拽、上下移动及其撤销/恢复均修改同一有序数组，每次变更仍调用物理核心重算。
  同一数组驱动可点击的 DUT→元件→50 Ω 信号链，避免拓扑图与实际计算顺序分叉。回损目标
  和端口频段来自手动会话/端口配置，前端在完整 sweep 上计算每频段最差点、平均值和目标裕量；
  手动页对频段的增删改同步回同一 `portConfigs` 和对应权重数组，并在反向、越界或重叠时
  停止通过/裕量判定。标准 Smith 网格直接使用后端复数 Γ；目标点阻抗按
  `Z = Z0(1+Γ)/(1-Γ)` 转换，回损目标圈半径为 `10^(-RL/20)`，点击图面仅选择最近的真实
  sweep 频点。这些是展示派生量，不替代后端返回的物理曲线与功率守恒结果。
  `/api/manual-refine/jobs` 在保持元件顺序、连接类型和端口端接不变的前提下执行确定性的
  对数坐标搜索：理想 L/C/R 优化标称值，直通线/Stub 优化特性阻抗与电长度，实测 S2P
  料号始终锁定。搜索阶段每个目标频段取 17 个物理频点并通过同一 `rfmatch_core` 计算，
  完成后再按 DUT 全部正频率点复算最终曲线；任务复用通用后台进度与取消协议，前端应用
  新参数前再次核验 DUT identity 和输入签名，并将旧网络压入撤销栈。网络、频段或门限变化
  会取消在途任务并清除过期优化摘要。最终连续参数还会各自执行 ±10% 的局部物理扰动，
  按目标评分影响量输出灵敏度排行；前端在结果摘要中展示高影响旋钮，并把提交任务前的完整
  sweep 作为临时“优化前”曲线叠加到 S11 与 Smith 图。该解释快照只属于当前网络，撤销或
  任意工程输入变化后立即清除，不会被误当作冻结方案或跨 DUT 证据。
  若优化后仍未满足全部频段，服务端同时记录每段最差频点的 Γ 与按当前 Z0 换算的复阻抗；
  随后分别在物理数组的 DUT 端和信号源端插入串联/并联理想 L、C，各以 9 个对数值调用同一
  核心粗探。只返回评分确有提升的前四个候选及预测最差回损，前端“应用”操作按返回的物理
  位置插入元件并进入正常撤销栈。该探针用于选择下一步拓扑方向，不是最终实测 BOM；应用后
  还会为前四个理想方向从活动目录按对数标称值距离选取最多 5 个不同料号，逐个以精确
  `part_number` 加载真实 S2P 并在相同频点复算。只有评分仍高于当前网络的料号才作为
  `measured_alternatives` 返回，按物理评分排序；单个损坏模型只计入诊断错误，不会中断整个
  优化任务。每个方向的粗筛前两名还必须在“DUT 全部正频率点＋17 点/频段搜索网格”的并集
  上复算，并以同一全频网格上的当前网络作为比较基线；返回项携带
  `verification=full_dut_plus_band_grid` 和实际点数，前端只将通过该二阶段验证的结果标成可应用。
  瓶颈频点也来自这份全频结果，而不是搜索网格的中间证据。前端可选择“应用理想”继续探索，或“应用实测”直接插入锁定料号；两者都会触发
  完整扫频和撤销记录。即使实测候选通过粗探，仍需后续全频验证和容差/良率分析。
  `/api/manual-yield/jobs` 把当前有序手动网络原样转换为 `rfmatch_core` 物理拓扑，并在目标频段
  的 DUT 原始点与 17 点/频段补充网格上运行 Monte Carlo。理想 L/C/R 与线段使用元件级或
  请求默认公差，实测 L/C 使用目录元数据公差并缩放其等效标称值而不直接缩放整张 S 参数矩阵；
  结果给出通过率、Wilson 置信区间、最差回损分位数、标称功率闭合和最差样本分支偏差。
  核心采样支持进度回调与协作取消；前端用 DUT identity 和网络签名拒绝过期结果，网络、频段
  或门限变化会清除旧良率证据。风险排序使用核心拓扑实际分支键映射，不能用显示模型名代替。
  命名冻结方案把精确实测料号、端接状态和指标快照写入受限的
  `extensions.manual_workspace` 项目扩展；扩展与主文档一起签名，HTML/PDF 仅报告
  冻结值，重新载入时仍要求已验证 DUT 并重算。新版快照同时保存频段、频段权重、回损
  门限、逐频段最差点/裕量与冻结时 DUT identity。对比矩阵按物理位置、连接方式、器件类型、
  精确料号或理想值计算相对当前网络的 BOM 增删改，并区分“冻结扫频”“当前 DUT 重算”与
  旧工程的“仅频点快照”；载入方案会原子恢复网络、端接、频段、权重和门限。
  S11/Smith 叠加最多选择 4 个冻结方案，仅持久化 ID，曲线为会话内懒加载物理重算缓存。
  DUT 会话键同时包含文件名和网络语义 SHA-256；原始文件 SHA 继续用于来源审计，但 CST
  仅改写注释/格式时不会清除手动缓存或优化会话。
- CST/HFSS 自动同步仍以 Touchstone 为可信边界。`/api/snp/watch/*` 为每个客户端建立
  当前目录基线，只有文件大小和 mtime 持续稳定并通过完整解析后才发布新修订；半写入、
  非单调频率或不完整矩阵不会替换当前 DUT。CST 直连在求解期间轮询结果树；同一工程的
  后续直出原子替换由项目路径 provenance 认领的稳定文件，并记录上一 SHA-256；若同名
  文件不属于该工程则递增命名，避免误覆盖。另以规范化频率、完整复数 S 矩阵和逐端口
  Z0 计算 `rfmatch.touchstone-network.v1` 指纹，区分 CST 导出文本变化与真实电气修订；
  语义未变时更新原始文件/来源记录但保留当前配置和手动工作区。CST 2025 直接接口见
  `docs/CST_INTEGRATION.md`。
- 前端把“应用数据路径”和“刷新 SNP 文件列表”作为两个不同事务：前者重建目录模型并
  发布新的数据源 revision，后者只读取文件清单，不得重置优化会话。隐藏工作区不预取
  实测元件或触发手动调谐；手动调谐、多场景和元件库面板仅在活动且目录 revision 就绪时加载，并对同一 revision
  去重。目录切换后，旧系列 ID 与新目录求交；无有效交集时回退到新目录默认系列。
- `/api/tuning/optimize` 的 `transmission_line` 模式使用同一物理核心自动搜索直通线、
  单 Stub，以及 line/Stub 在连接器侧或 DUT 侧的组合。搜索由后台 job 执行，支持进度、
  软超时、评估上限和协作取消；选中候选后可在原始 SNP 点或密集频点精确重算。
- 启用 `transmission_line.microstrip.enabled` 后，搜索中的每个电气候选都会受制造线宽
  限制并转换为 `MicrostripLineModel`，随后以频率相关 Z0、有效介电常数、铜损和介质
  损耗重新评分。良率路径显式扰动线宽、线长、板厚和介电常数，且线宽公差采用
  Optenni `msWidthToler` 语义。模型范围与排除项见 `docs/PCB_MICROSTRIP_MODEL.md`。
- 通用真实元件搜索的单端口和多端口路径都把 API 软时间预算传入核心协作取消检查。
- 组件目录分为“完整可发现目录”和“默认校准目录”。当前本机 Optenni 文件夹扫描识别
  16,480 个模型、94 个文件夹级器件族；默认仍使用 0402HP/GQM18。Web 传入带类型的
  `L::family / C::family` 标识，API 为单次请求构建只读过滤目录，不改变全局默认。
  系列过滤后再执行厂商、封装、最大容差、额定电压代码和介质约束。SQLite 中的容差/
  封装字段标记为 `database`，由料号解析得到的电压代码/介质标记为 `part_number_inferred`，目录
  推导字段标记为 `catalog_path`，无法验证的字段标记为 `unknown`；用户可保留未知项
  或严格排除。预览 API 与实际搜索共用同一筛选函数。候选诊断记录配置、来源统计、
  L/C 数量与目录身份 SHA-256；更换数据目录会清除运行时检查点，
  避免跨目录复用旧模型。
- 单端口权威求解入口是 `run_tuning_single`。统一产品 API 和兼容 `/api/tune/single` 均委托
  此函数；兼容路由只保留协议字段。Web 不再暴露 `/api/optimize`、`/api/multipass`、
  `/api/joint-optimize` 或旧 tune 调用，避免新功能误接入历史求解器。核心同时保留向后兼容的
  `allowed_topology_codes` 和产品使用的 `allowed_topology_codes_by_port`；逐端口项优先于全局项。
  白名单在理想种子、深层发现和真实料号展开前生效，而不是在 API 返回后过滤，因而被排除
  的网络不会消耗最终排名预算。`0` 是显式裸 DUT 拓扑；省略白名单表示自动搜索。
- `max_components=0` 仍通过相同核心构造 `Problem` 并执行全频带裸 DUT 功率求解；该路径
  不依赖元件目录，也不回退 legacy。结果将 bare-DUT 与 measured-component provenance
  分开，报告不会把“没有元件”描述成实测元件综合。
- 端口配置中的 `port_weight` 与等长 `band_weights` 在 API 边界归一化，二者乘积写入核心
  `Band.weight`。该权重是 dB 目标 margin 的工程优先级乘数，不是概率；默认 `1.0` 保持旧
  评分。连续拓扑种子和最终实测 S2P 搜索必须使用同一有效权重，自定义权重禁止静默回退到
  不支持该契约的 legacy evaluator。候选诊断、项目及 HTML/PDF 报告保存原始和有效权重。
- 单端口和多端口 measured core 都从实际非空目录推导 `available_component_kinds`，理想
  种子和深层拓扑只生成可由当前目录实现的器件类型。因此只有 L 的目录仍可综合纯电感
  网络，缺少 C 不会触发整体 legacy 回退；全端口零元件请求无需目录即可产生包含完整
  耦合矩阵的全带物理基线。检查点也把可用类型集合纳入理想前沿失效条件。
- 明确逐端口拓扑、每端口最多两元件的 thorough 联合搜索增加两级耦合桥接：先在完整 DUT
  矩阵上同时优化全部端口的理想元件值，再以端口为块联合替换同一端口的两个真实料号。
  块搜索保留一个小 beam，使暂时较差但能与其它端口协同的移动不会被单坐标贪心提前删除。
  实测邻域按不同标称值选取，并在每个值内保留多个 S2P 型号；产品目录因此按料号/模型身份
  去重，不再按标称值删除物理不同的可采购器件。
- 产品 `engine.touchstone.parse_touchstone` 通过 `rfmatch_core.parse_touchstone_text` 构造
  运行时对象，Touchstone 的三端口以上行优先矩阵顺序、二端口显式顺序、Full/Lower/Upper、
  RI/MA/DB 转换、标量/逐端口正实数参考阻抗和
  严格校验只有一个权威实现。核心 S↔Z、理想元件与节点物理求解均保留向量 Z0；上传
  失败返回 HTTP 400 且保留当前 DUT，不会用半解析数据覆盖活动会话。复数参考阻抗仍
  明确拒绝，避免在尚未认证 Kurokawa power-wave 口径时给出错误功率结果。
- 元件详情接口返回字段级来源和模型身份；替代料分析先按类型、目录约束和标称值窗口
  有界预选，再在请求频带对完整二端口 S 矩阵计算 RMS/最大复数差异。最多物理比较
  500 个候选并明确报告截断/失败，避免无界加载完整目录。项目 BOM CSV 按相同料号与
  采购属性聚合，同时保留 `P端口:位置`，因此数量统计不会丢失网络位置可追溯性。
- Web 的单端口和多端口主流程不再调用长连接同步优化，而是统一提交 `/api/tuning/jobs`。
  核心逐端口报告 `ideal_search → measured_expansion → joint → refine`，API 线程执行避免阻塞
  事件循环。用户取消后，任务状态为 `cancelled`，但若核心已完成基线或精确候选，结果会随
  job 保留并在 UI 中展示，不把已付费的物理计算丢弃。
  单端口 `_per_port` 直接复用完整问题缓存；截止发生在离散展开时，已完成的候选直接进入
  最终排名。5–6 元件使用固定确定性拓扑主干，额外预算只扩大离散拓扑邻域和精修集合，
  使长预算成为短预算的搜索超集。
- 多场景页面通过 `/api/multi-scenario/jobs` 提交共享网络搜索，并复用同一任务状态与取消路由。
  `MultiScenarioOptimizer` 在理想拓扑预筛、物理粗筛、完整目录精修和独立密集验证中设置协作式
  取消点；进度携带阶段、拓扑序号、耗时和物理评估数。兼容入口 `/api/multi-scenario/optimize`
  仍保留，但 CPU 工作通过线程执行，不再阻塞 API 事件循环。
- `/api/tuning/continue` 读取当前 live session 的最近请求并增加总预算。单端口与多端口联合实测 S2P
  搜索复用运行时优化器检查点：已加载模型、理想拓扑前沿和精确物理评估缓存都不重复，
  本轮截止只使用用户新增的秒数。旧/新候选仍按网络签名合并再排序，因此结果单调不退化。
  多端口检查点还保留逐端口子问题对象，使端口候选和全矩阵联合候选都能命中同一精确缓存；
  任一端口的拓扑白名单变化会清空理想候选前沿，但仍保留与网络签名一致的精确物理缓存。
  不具备检查点的兼容路径明确标记为 `deterministic_rerun_merge`；项目快照不序列化含 NumPy
  数组和模型句柄的运行时对象，需先重新计算才能续跑。
- Web 通过 `/api/tuning/continue/jobs` 提交追加预算；它与首次优化共享同一任务状态、进度、
  取消和部分结果契约，避免检查点续算重新变成长 HTTP 请求。
- `engine/search_quality.py` 是产品搜索档位与执行计划的唯一契约。命名档位在 Pydantic 请求
  验证时权威设置 timeout、输出深度和带内采样数；`auto/custom` 保留调用方数字并按实际预算
  推断显示级别。计划同时判断自动拓扑超深度与固定拓扑耦合精修的资格，返回预算、策略、适用
  边界和参考基准说明。后台 job 在每次进度更新中附加 elapsed/budget/fraction，终态保留这些
  元数据；续算显式转为 custom 累计预算，不能重新套用原预设覆盖总秒数。
- `engine/calibration_evidence.py` 从 `artifacts/benchmarks` 的权威 JSON 动态构建 recall、
  saved-winner 排名、跨软件误差和参考耗时声明，不在服务代码中复制数字。每个主/子证据
  都附带运行时 SHA-256；缺失、无效 JSON 或不支持的 schema 会失败关闭。产品可通过
  `/api/calibration/status` 在优化前显示单端口和多端口校准工件的实际哈希。
- 多场景搜索保持一个共享物理网络跨全部 DUT 状态的核心语义，但搜索
  已从按标准拓扑顺序逐个耗尽预算改为三层调度：连续理想 LC 对全部拓扑分配优先级、每个
  拓扑取得小型实测目录粗筛证据、高分值再回到完整唯一值目录做邻域精修。搜索结果不会直接
  输出；独立高密度频率网格复算并重新排序后才返回，同时分别报告搜索预算和验证开销。
- `transmission_line.layout_blocks` 把 SNP 目录内的二端口 EM/VNA 数据转换为固定
  `S2PModel`，按 DUT 向连接器的顺序放在可综合网络两侧。服务层禁止频率外推并计算
  被动性/互易性诊断；核心支持二端口方向交换以及不经过奇异 Z 矩阵的等实数功率波
  重归一化，并以带 S21/条件数保护的链矩阵执行左右 fixture 去嵌入和重级联残差验证；
  项目层保存布局与 fixture 的 SHA-256 和变换设置，恢复和报告使用同一依赖清单。契约与
  参考面限制见 `docs/LAYOUT_BLOCKS.md`。
- `data/Murata`、`data/snp`：运行数据，不进入 Git。

## 目标演进

1. 以 `packages/rfmatch-core` 的 S/Z/Y、物理功率守恒和 golden 测试作为唯一数值真源。
   单端口通用 L/C 连续综合使用等价的向量化有损功率传播；多端口和任意 S2P 仍回退
   到统一节点 Y 求解，以兼顾交互速度与模型通用性。
2. Touchstone 已通过 adapter 收敛到独立核心；继续替换残余网络计算、评分和搜索，每一步做双跑比对。
3. 将 `server.py` 按 `data`、`components`、`tuning`、`legacy` 四个 router 拆分，并通过 service 注入 `AppState`。
4. 将同步长任务迁移为可取消的 job：提交、进度、结果、取消；前端不再依赖单个长 HTTP 请求。
5. 删除完成迁移的 `/optimize`、`/multipass`、`/joint-optimize`、`/tune/*`，只保留 `/tuning/*` 的版本化 API。

## 配置

默认数据目录相对仓库根目录解析，可用环境变量覆盖：

- `RFMATCH_SNP_DIR`
- `RFMATCH_MURATA_DIR`
- `RFMATCH_DB_PATH`
- `RFMATCH_OPTENNI_COMPONENT_DIR`
- `RFMATCH_PROJECTS_DIR`
- `RFMATCH_HOST`（直接运行 `server.py` 时使用）

默认仅监听 `127.0.0.1`。如果未来允许局域网访问，需要先增加身份认证、每用户会话和文件访问白名单。

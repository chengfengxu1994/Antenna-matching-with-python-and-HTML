# RF Matching 项目保存格式

项目文件默认保存在 `artifacts/projects/*.rfmatch.json`，也可通过
`RFMATCH_PROJECTS_DIR` 指向其他目录。界面顶部的 **Projects** 提供保存、列表、打开和
**HTML report** 与 **PDF report** 下载入口。

## 保存内容

- 格式版本、项目 ID、名称、创建/更新时间；
- API、`rfmatch-core` 和 Python 版本；
- Touchstone 相对文件名、SHA-256、端口数、频率范围、参考阻抗；
- 完整的统一调优请求，包括端口、频段、目标、命名搜索质量档位、搜索宽度、带内点数和超时；
- 元件数据库 SHA-256，或版本化的 S2P 内容清单指纹；清单覆盖元件身份、工程元数据与每个
  独立文件/ZIP 条目的源字节，不包含本机绝对目录；
- 效率数据来源；
- 全部候选方案、当前选择、逐端口指标、曲线和功率平衡快照。
- `extensions.manual_workspace` 中的手动调谐工作区 v1：活动激励端口、目标频率、
  逐端口工作网络，以及最多 12 个命名冻结方案的端接状态、精确元件和 RL/VSWR/
  输入阻抗/功率误差快照，以及最多 4 个当前曲线叠加方案 ID。该扩展受同一文档
  SHA-256 完整性覆盖。工作网络和方案中的元件数组为有序物理级联，顺序按 DUT 向
  信号源排列；保存、迁移和恢复不得把它当集合排序。

候选 `search_diagnostics.search_plan` 另外保存实际执行策略、累计预算、自动拓扑/受约束耦合
资格和参考校准说明。命名档位是可复现参数模板，不是当前 DUT 的最优性保证；报告会同时显示
这一限制。追加预算后的请求保存为 `custom`，并在 continuation 元数据保留原始档位。

每个文件包含对规范化 JSON 文档计算的 SHA-256。它用于发现写入中断、手工修改和意外损坏，
不是数字签名，也不能对抗能够同时修改正文和摘要的攻击者。写入采用同目录临时文件、`fsync`
和原子替换，避免应用退出时留下半个项目文件。

当前项目 Schema 为 v2，并使用固定的 `project_format=rfmatch.project` 和保留的
`extensions` 命名空间。v1 文件仍可打开：系统先按 v1 原文验证 SHA-256，再确定性迁移到 v2，
并在 `migration_history` 中记录来源版本和原始摘要。只读打开不会静默改写磁盘；用户下次保存时
才以 v2 原子覆盖。未知的未来版本会明确拒绝，避免按旧结构猜测字段含义。
手动工作区扩展有独立 `schema_version=1`、强类型 API 校验、端口/元件/方案数上限和
方案 ID 唯一性检查；不会把未限制的界面状态直接塞入工程 JSON。

## 打开语义

打开项目时必须先验证项目摘要，然后在当前 `RFMATCH_SNP_DIR` 下定位原始 Touchstone，并核对
SHA-256。默认情况下文件缺失或内容变化都会拒绝恢复，防止旧结果错误绑定到同名的新测量数据。

恢复后的候选、指标和已保存曲线属于 `snapshot` 模式，可立即查看和比较。运行时的元件对象不会从
JSON 反序列化，以免不可信项目文件构造任意对象。因此精确密集扫频、功率重新计算和继续优化需要
使用保存的 `tuning_request` 与已验证的元件库重新运行；界面和 `/api/tuning/status` 会明确显示这一限制。
冻结手动方案的指标同样是快照；原始 Touchstone 验证通过时，界面载入方案后会经
`/api/manual-tune` 用当前物理核心重算，而不会把冻结指标当成新计算结果。
曲线叠加也只保存方案 ID，不存储易过期的曲线；恢复工程后会对每个已选方案懒加载重算。
同名 Touchstone 若 SHA-256 变化，手动会话和曲线缓存立即失效，避免跨 CST 求解修订比较。

跨设备导入时可以选择“仅查看快照”。“查找并修复”会在当前 SNP 数据目录中为 DUT、布局块与
去嵌夹具寻找 SHA-256 完全一致的文件；匹配路径记录在 `extensions.file_relinks`，不会抹掉快照内
的原始文件名。只有 DUT、全部依赖以及元件数据库/`rfmatch.component-catalog.v3` 内容清单同时
验证通过，产品才会启用“按保存配置重新计算”。旧 S2P 工程若没有可移植内容清单，仍可审阅，但
必须在原环境验证后另存新快照才能建立这一证明链。

## API

- `GET /api/projects`：列出项目、当前 Schema 和迁移来源；损坏文件不会被隐藏，而以
  `invalid` 状态展示。
- `POST /api/projects/save`：保存新项目，或传入已有 `project_id` 更新。
- `POST /api/projects/load`：验证并恢复安全快照。
- `POST /api/projects/import`：验证外部工程 JSON；同 ID 冲突默认创建带来源记录的副本。
- `POST /api/projects/relink`：扫描并记录内容完全一致的 DUT、布局与夹具替代路径。
- `GET /api/projects/{project_id}/snapshot.json`：导出带完整性摘要的工程 JSON。
- `GET /api/projects/{project_id}/report`：从验证通过的快照生成自包含 HTML；默认以附件
  下载，`?download=false` 用于浏览器预览。
- `GET /api/projects/{project_id}/report.pdf`：生成原生分页 PDF；默认下载，
  `?download=false` 时以内嵌方式打开。

## 报告语义

HTML 与 PDF 报告格式分别使用独立 Schema。两者均包含项目/输入 SHA-256、软件与核心版本、
全部自动候选摘要、冻结手动方案对比、当前选择、BOM、可调状态映射、隔离目标、完整功率口径及保存的逐频曲线。
跨频段之间不会绘制虚假的连接线。HTML 不加载外部脚本、字体或网络资源；原生 PDF 使用嵌入字体、
A4 自动分页、重复表头、页码和完整性摘要，可直接归档或打印。

当前没有删除 API，避免误删用户项目。需要清理时可在明确确认后管理项目目录。

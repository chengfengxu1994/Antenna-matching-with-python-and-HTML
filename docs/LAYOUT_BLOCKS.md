# EM / VNA S2P 布局块

传输线自动综合可以把连接器、焊盘、过孔、封装或一段已仿真走线的二端口
Touchstone 文件作为固定物理网络参与每一次候选评分。这不是结果后的插损修正：布局块与
line / Stub 按顺序级联，完整 S 参数的失配、相位、损耗和频率响应都会改变最优解。

## 使用方法

1. 把 `.s2p` 文件放入配置的 SNP 目录（默认 `data/snp`）。界面不会读取该目录之外的路径。
2. 在传输线自动综合的 **Fixed EM/VNA layout blocks** 中添加文件。
3. 选择位置：
   - `DUT side`：布局块位于 DUT 与待综合传输线网络之间；
   - `Connector side`：布局块位于待综合网络与 50 Ω 系统端口之间。
4. 同一侧有多个布局块时，界面从上到下即为从 DUT 向连接器的级联顺序。
5. 选择被动性策略：`Warn` 保留并标记疑似非被动数据，`Reject` 在优化前拒绝它。
6. 如果文件端口 2 朝向 DUT，启用 `Flip 1↔2`；这会交换 S11/S22 与 S21/S12，不能
   仅靠改变布局块所在侧来替代。
7. `Native Z0` 保留 Touchstone 的参考阻抗；`System Z0` 先把功率波 S 参数重归一化到
   当前 DUT/系统阻抗。物理网络理论上不变，但统一参考阻抗便于导出、比较和诊断。
8. 如果布局文件包含测试夹具，选择可选的 Left / Right fixture。定义严格为
   `measured = left → target layout → right`；fixture 自身也可独立 `Flip 1↔2`。

## 输入契约与可追溯性

- 目前只接受二端口 `.s2p`；参考阻抗使用文件中的 `R` 值。
- S2P 必须覆盖所有启用频段采样点。仅容忍端点浮点换算误差，不执行静默外推。
- 导入时检查有限值、严格递增频率、最大奇异值、最大功率增益超量和互易误差；诊断
  同时记录原生/有效参考阻抗、端口方向与是否重归一化。
- 结果记录文件名、位置、SHA-256 和被动性诊断；HTML/PDF 报告列出输入依赖。
- 保存项目后再次打开，会重新核对布局文件的 SHA-256。文件缺失或内容变化时，完整性
  校验返回冲突，防止旧结果被误认为可复现。
- 去嵌入前 fixture 会转换到布局块的有效 Z0，并插值到布局频率；任何覆盖不足都拒绝。
  链矩阵 `|S21|` 过小或条件数超过 `1e10` 时停止，结果记录条件数和重级联残差。

## 当前边界

当前版本支持左右二端口 fixture 去嵌入、端口翻转和等实数参考阻抗重归一化，但不支持
TRL/SOLT 校准求解、gating、复数/逐端口 Z0、混合模转换、因果性修复或被动性强制。
去嵌入本身会放大测量噪声；条件数合格不等于校准参考面正确。对不连续结构优先使用经过
端口校准的 EM/VNA S2P；对普通均匀微带仍可
使用内建 PCB 几何模型。最终投板前应以完整板级 EM 或实测再次验证。

## 回归基线

`scripts/run_layout_block_benchmark.py` 使用一个互易、匹配、0.7 dB 插损的合成二端口，
同时验证 DUT 侧/连接器侧的顺序、逐频传递功率、器件损耗与功率闭合；另用非互易矩阵
验证端口翻转、50↔75 Ω 重归一化往返精度，以及 `left → DUT → right` 合成测量的恢复
误差、fixture 条件数和重级联残差。确定性产物为
`artifacts/benchmarks/layout-block-physical-baseline.json`。
